iPython Notebook, Django and Vagrant

16 December 2014

iPython is a great tool for developing and debugging applications in python. I use it heavily at work for quickly prototyping ideas but also as a way to interact with models in a django app.

I was tasked with generating some statistics about our app at work. However the next day I was asked to turn these commands into a Django managment command that could be run again on demand as well as share the commands I used to generate the analysis. Great!

But there was a problem, what commands did I run? I hadn't saved anything from when I ran the analysis.

iPython Notebook

I have used iPython Notebook before, using it at PyconUK to help predicate survivors of the titanic. It was the perfect way to record all of my analysis and commands. But I ran into a few problems getting it setup and intergrated into my workflow.

I use vagrant to manage the development VM. Some tweaking was needed to get iPython Notebook working correctly.

Install and upgrade django_extensions
pip install django_extensions --upgrade
Install all of the dependencies for iPython Notebook.
 pip install jinja2
 pip install pyzmq 
 pip install Tornado
Install iPython Notebook
 pip install "ipython[notebook]"
Add these lines to your settings.py
IPYTHON_ARGUMENTS = [      'django_extensions.management.notebook_extension',  
'--ext',  
'myproject.notebook_extension',  
'--ip',  
'*',]
Enable Port Forwarding in VagrantFile
config.vm.network "forwarded_port", guest: 8888, host: 8888

That is it, you can now can to http://localhost:8888/
and use iPython Notebook with your Django App on your Vagrant Machine
